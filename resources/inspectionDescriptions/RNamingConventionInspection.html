Reports inappropriate property names.

Variable and function names should use only lowercase letters, numbers, and underscores. Use underscores (so-called snake case) to separate words within a name. Also, in function names, use dots in S3 methods.

See the tidyverse style guide for more details.

Example:

camelCaseVariable <- 10

When you apply the quick-fix, this code changes to:

camel_case_variable <- 10
